# SavvyDealer Static Site

Static HTML site for savvydealer.com. Python generators produce all HTML, served by Express on Cloud Run.

## Quick Start

```bash
git pull                     # always pull first
python generate.py           # regenerate all HTML (51 pages)
npm run build:css            # rebuild Tailwind CSS
npm start                    # test at http://localhost:8080
```

Or all at once: `npm run build && npm start`

## Architecture

```
generate.py              # Main generator: shared templates (head, nav, footer, CTA, FAQ)
generate_pages.py        # Page generators: home, services, about, book, products, etc.
generate_blog.py         # Blog: parses content/blog/*.md, generates listing + 26 post pages
generate_resources.py    # Resource guides: SEO, GEO, Facebook Ads, Reputation
server.js                # Express: security headers, form APIs, static file serving
src/input.css            # Tailwind v4 config (fonts, theme)
content/blog/            # Blog posts as Markdown with YAML frontmatter
public/                  # Generated output (HTML, CSS, JS, images) - DO NOT hand-edit HTML
public/images/           # All site images (PNG, WebP, JPG)
public/js/               # Client JS (forms.js, nav.js, lead-modal.js)
```

## Key Rules

- **Never hand-edit files in `public/` directly** (except `public/js/` and `public/images/`). All HTML is generated by Python. Edit the generators instead, then run `python generate.py`.
- **Always run `npm run build:css` after changing HTML** so Tailwind picks up new utility classes.
- **Pull before starting work.** Push when done. This prevents merge conflicts when multiple people edit.
- Blog posts live in `content/blog/` as Markdown. Edit those files, then regenerate.

## How Pages Are Generated

`generate.py` defines shared templates and constants:
- `head()` - HTML head with meta tags, OG, schema.org, GTM, Tailwind CSS link
- `nav()` - Sticky header with Products dropdown, mobile menu
- `footer()` - Site footer with links, social, legal
- `cta_section()` - "Ready to Dominate?" call-to-action block
- `faq_section()` - FAQ accordion with schema.org markup
- `write_page()` - Writes HTML to `public/`

`generate_pages.py` has individual page generators:
- `service_page(data)` - Template for all 6 service pages (uses SERVICE_PAGES dict)
- `generate_home()` - Homepage with hero, product cards, features, testimonials
- `generate_about()`, `generate_products()`, `generate_book()`, etc.

`generate.py:main()` calls everything and also generates `sitemap.xml`, `robots.txt`, `llms.txt`.

| Primary color | `#0088ff` |
| Heading font | Space Grotesk (`font-serif` in Tailwind) |
| Body font | Inter (`font-sans`) |

## Common Tasks

### Edit a service page (e.g., Facebook Ads)
1. Open `generate_pages.py`, find the entry in `SERVICE_PAGES` dict
2. Edit the content (title, description, benefits, features, FAQ)
3. Run `python generate.py && npm run build:css`

### Add a blog post
1. Create `content/blog/your-post-slug.md` with YAML frontmatter:
   ```yaml
   ---
   title: "Post Title"
   description: "Meta description"
   author: "Author Name"
   date: "2026-03-05"
   category: "SEO"
   featuredImage: "/images/your-image.png"
   ---
   Post content in Markdown...
   ```
2. Add the image to `public/images/`
3. Run `python generate.py && npm run build:css`

### Edit shared layout (nav, footer, CTA)
Edit the corresponding function in `generate.py`, then regenerate.

### Add a new image
Drop it in `public/images/`. Reference it as `/images/filename.png` in generators.

## Server (server.js)

- Express with compression, security headers (CSP, HSTS, etc.)
- Form API routes: `/api/lead-form`, `/api/quick-text`, `/api/schedule-demo`, `/api/indie-demo-request`
- Forms use Resend for email, reCAPTCHA for spam, honeypot fields, token validation
- Legacy URL redirects from the old React site
- Clean URLs: `/about` serves `about.html`

## Prerequisites

- Node.js 20+
- Python 3.10+ with `markdown` and `pyyaml` packages
- `npm install` for Express dependencies
- `npm install --include=dev` if you need to rebuild Tailwind CSS

## Tailwind CSS (v4)

Config is in `src/input.css` (NOT a `tailwind.config.js` -- v4 uses CSS-based config). The `@source` directive tells Tailwind to scan `public/**/*.html` for utility classes. Output goes to `public/css/tailwind.min.css`.

Do NOT use the Tailwind CDN. CSP headers block it. Always use the compiled CSS file.
